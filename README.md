# [기말 프로젝트] MJ-Maker

**MJ-Maker**는 마이클 잭슨 앨범 이미지를 떨어뜨려서 같은 앨범끼리 합치는 물리 기반 머지 퍼즐 게임입니다.
기본 규칙은 수박게임류와 비슷하지만, 앨범 LP판, 회전 연출, 효과음, 댄스 밈 요소를 섞어서 마이클 잭슨 테마로 만들었습니다.

한국공학대학교 스마트폰게임프로그래밍 기말 프로젝트로 제작했습니다.

## 1. 게임 소개

MJ-Maker는 위에서 앨범 LP판을 떨어뜨리고, 같은 레벨의 LP판이 닿으면 다음 단계 앨범으로 합쳐지는 게임입니다.
LP판을 최대한 많이 합쳐서 점수를 올리는 것이 목표이고, 오브젝트가 화면 위쪽 데드라인을 일정 시간 넘으면 게임이 끝납니다.

처음에는 단순히 수박게임 구조를 따라가는 정도로 생각했지만, 그냥 과일만 앨범으로 바꾼 게임이면 재미가 약할 것 같았습니다.
그래서 LP판 회전, 마이클 잭슨 효과음, 댄스 자세처럼 보이는 오브젝트 변형, 충돌 시 배경 플래시 같은 요소를 추가했습니다.

주요 특징은 다음과 같습니다.

* 마이클 잭슨 앨범 이미지를 단계별 오브젝트로 사용
* LP판이 굴러가듯 회전하는 물리 연출
* 같은 앨범끼리 합쳐지는 머지 시스템
* 충돌과 머지 상황에 맞는 효과음
* 데드라인 경고 및 게임오버 처리
* 최고점수 저장
* 일부 오브젝트의 댄스 밈 형태 변화
* 충돌 강도에 따른 배경 플래시 연출

### 목표 변경 사항

처음 목표였던 앨범 머지 게임 구조 자체는 크게 바꾸지 않았습니다.
대신 만들다 보니 생각보다 물리 충돌 쪽에서 문제가 많이 생겼습니다.

처음에는 단순 원형 충돌로 충분할 줄 알았는데, 이미지 모양이랑 실제 충돌 범위가 안 맞아서 게임이 어색했습니다.
그래서 후반에는 단순히 콘텐츠를 늘리기보다 충돌 처리, 회전, 최적화 쪽에 시간을 더 많이 썼습니다.

추가로 구현한 항목은 다음과 같습니다.

* 정지한 오브젝트를 계산에서 제외하는 Sleeping 처리
* 이미지 알파값 기반 멀티서클 히트박스
* 충돌 강도 기반 배경 플래시
* 최고점수 저장
* 데드라인 경고 연출
* 볼륨 토글 및 일시정지

## 4. 사용된 기술

### Android / Kotlin

* Kotlin 기반 게임 로직 작성
* Android Canvas를 이용한 2D 렌더링
* MediaPlayer를 이용한 BGM 재생
* SoundPool을 이용한 효과음 재생
* SharedPreferences를 이용한 최고점수 저장

### 게임 루프 / 렌더링

* Choreographer 기반 프레임 콜백 사용
* delta time 기반 업데이트
* Canvas Matrix 변환을 이용한 가상 해상도 처리
* Layer 구조를 이용해 배경, 오브젝트, UI를 분리해서 렌더링

### 물리 처리

* 중력 기반 낙하
* 벽 / 바닥 충돌
* 오브젝트 간 충돌 판정
* 충돌 후 위치 보정
* 속도와 반발계수 기반 충돌 반응
* 각속도와 마찰을 이용한 회전 처리
* Sleeping State를 이용한 불필요한 물리 계산 감소

### 충돌 최적화

* Bounding Circle로 1차 충돌 검사
* 이미지 알파값을 이용한 멀티서클 히트박스 생성
* 생성된 히트박스 캐싱
* 정지 상태의 오브젝트 계산 제외

## 6. 수업 내용에서 차용한 것

### GameView 루프

`Choreographer.FrameCallback`을 이용해서 매 프레임 update와 draw가 호출되도록 구성했습니다.
프레임 시간 차이를 이용해서 기기 성능이 달라도 게임 속도가 심하게 달라지지 않게 처리했습니다.

### SceneStack

TitleScene, MainScene, ResultScene처럼 화면을 나눠서 관리했습니다.
일시정지 화면도 별도 상태로 처리했습니다.

### World / Layer 구조

게임 오브젝트를 레이어별로 나누어 관리했습니다.

* Background Layer
* Object Layer
* Effect Layer
* UI Layer

이렇게 나누니까 배경, 오브젝트, UI가 그려지는 순서를 관리하기 편했습니다.

### GameMetrics

기기 해상도가 달라도 같은 위치와 크기로 보이도록 900x1600 가상 해상도를 기준으로 좌표를 변환했습니다.

### BitmapPool

이미지를 매번 새로 불러오지 않고 캐싱해서 사용했습니다.
이미지 로딩과 메모리 할당 때문에 생길 수 있는 프레임 드랍을 줄이기 위해 사용했습니다.

## 7. 직접 개발한 것

### 앨범 머지 로직

같은 레벨의 LP판이 충돌하면 다음 레벨의 LP판으로 합쳐지도록 만들었습니다.
합쳐질 때 기존 오브젝트를 제거하고, 그 위치에 다음 레벨 오브젝트를 생성합니다.
점수도 이 타이밍에 같이 증가합니다.

### 이미지 기반 멀티서클 히트박스

처음에는 오브젝트 하나당 원 하나로 충돌을 처리했습니다.
그런데 이미지 형태와 실제 충돌 범위가 너무 안 맞아서, 캐릭터나 앨범 이미지가 이상하게 부딪히는 문제가 있었습니다.

그래서 이미지의 알파 채널을 검사해서 불투명한 부분에 여러 개의 충돌 원을 배치하는 방식으로 바꿨습니다.
이 방식 덕분에 이미지 모양에 좀 더 가까운 충돌 처리가 가능했습니다.

### 회전 물리

충돌할 때 오브젝트가 그냥 밀리기만 하면 LP판 느낌이 약했습니다.
그래서 충돌 위치와 속도에 따라 각속도가 생기도록 만들었습니다.
완전한 물리 엔진 수준은 아니지만, LP판이 부딪히고 굴러가는 느낌은 낼 수 있었습니다.

### 댄스 밈 연출

일부 오브젝트는 단순한 LP판이 아니라 마이클 잭슨의 자세나 밈처럼 보이도록 만들었습니다.
렌더링 크기만 바꾸면 실제 충돌 범위와 달라지기 때문에, 시각적 변화와 충돌체 변화가 같이 움직이도록 처리했습니다.

### 데드라인 경고

오브젝트가 상단 데드라인을 넘었다고 바로 게임오버가 되면 너무 갑작스럽게 느껴졌습니다.
그래서 일정 시간 경고를 보여주고, 그 시간이 누적되면 게임오버가 되도록 만들었습니다.

### 사운드 / 볼륨 처리

BGM과 효과음을 분리해서 관리했습니다.
BGM은 MediaPlayer, 효과음은 SoundPool을 사용했습니다.
볼륨 토글 상태도 저장해서 다시 실행해도 상태가 유지되도록 했습니다.

### 최고점수 저장

게임 종료 시 현재 점수가 최고점수보다 높으면 SharedPreferences에 저장합니다.
결과 화면에서는 현재 점수와 최고점수를 같이 보여줍니다.

**Q. 조작은 어려웠어?**  
A. 쉬었다.

**Q. 가장 괜찮았던 부분은?**

**Q. 아쉬웠던 부분은?**

* 지인 평가 점수: 3.5 / 10

## 10. 아쉬운 점

### 하고 싶었지만 못 한 것

원래는 특정 레벨에 도달할 때마다 마이클 잭슨 곡 분위기에 맞춰 배경이나 회전 속도가 바뀌는 연출을 넣고 싶었습니다.
하지만 물리 처리와 충돌 안정화에 시간이 꽤 많이 들어가서, 음악 연동 연출까지는 완성하지 못했습니다.

### 판매를 위해 보충할 것

앱을 실제로 스토어에 올린다면 지금 상태로는 부족한 부분이 많습니다.
특히 아래 요소는 추가로 필요하다고 봅니다.

* 튜토리얼
* 난이도 조절
* 랭킹 시스템
* 더 많은 스킨과 배경
* 기기별 성능 테스트
* 저작권 문제가 없는 리소스로 교체
* 광고 또는 과금 구조 검토

### 해결하지 못한 문제 / 버그

오브젝트가 좁은 공간에 많이 몰릴 때, 충돌 보정이 겹치면서 일부 오브젝트가 떨리는 현상이 가끔 발생합니다.
Sleeping 처리와 충돌 보정으로 대부분 줄이긴 했지만, 아주 복잡하게 쌓인 상황에서는 완전히 없애지는 못했습니다.

### 프로젝트 중 어려웠던 점

가장 어려웠던 부분은 충돌 처리였습니다.
처음에는 단순 원형 충돌만 사용했는데, 이미지 형태와 실제 충돌 범위가 맞지 않아서 게임이 어색하게 느껴졌습니다.
그래서 이미지 알파값을 검사해서 여러 개의 충돌 원을 만드는 방식으로 바꿨습니다.

또 오브젝트가 많아질수록 충돌 검사량이 늘어나는 문제도 있었습니다.
이를 줄이기 위해 Bounding Circle 검사, 히트박스 캐싱, Sleeping State를 추가했습니다.

## 11. 수업에 대한 의견

### 이번 수업에서 기대한 것

스마트폰에서 실제로 실행되는 게임을 만들고, APK로 빌드해서 제출하는 과정을 직접 해보고 싶었습니다.
예제만 따라가는 게 아니라, 하나의 완성된 게임 형태까지 만들어보는 것을 기대했습니다.

### 얻은 것

이번 프로젝트를 하면서 Android 환경에서 게임 루프, Scene 전환, Canvas 렌더링, 사운드 재생, APK 빌드 과정을 직접 경험할 수 있었습니다.
특히 a2dg 구조를 사용하면서 Scene과 Layer를 나누는 방식이 실제 프로젝트에서 왜 필요한지 알 수 있었습니다.

### 얻지 못한 것

스토어 출시나 서비스 수준의 최적화까지는 다루지 못했습니다.
다양한 해상도와 기기에서 테스트하는 방법도 더 깊게 해보지는 못했습니다.

### 더 좋은 수업이 되기 위해 변화할 점

APK 서명, 기기 설치, 발표 영상 구성처럼 제출 직전에 문제가 생기기 쉬운 부분은 예시 자료가 조금 더 있으면 좋을 것 같습니다.
중간 제출마다 README 예시나 체크리스트가 있으면, 최종 제출 때 요구사항을 빠뜨리는 일이 줄어들 것 같습니다.
