## 2. 게임 소개
마이클 잭슨의 연대기별 사진 객체를 낙하시켜 동일한 사진끼리 합치며 진화시키는 물리 기반 머지 퍼즐 게임입니다. 사진이 합쳐질수록 마이클 잭슨의 커리어가 최신화되며, 화면 상단까지 사진이 쌓이지 않도록 관리하는 것이 핵심입니다.

## 6. 게임 구조 및 설계

### Activity 구성
* **MainActivity**: 앱의 엔트리 포인트. `ActivityMainBinding`을 통해 UI 레이아웃을 로드하며, `Scene.push(TitleScene())`을 호출하여 게임 엔진을 구동합니다.

### Scene 구성 및 전환
* **TitleScene**: 게임 시작 화면. 터치 시 `MainScene`으로 전환(`push`).
* **MainScene**: 메인 게임 루프. 객체 투하, 머지, 물리 연산이 이루어지는 핵심 장면.
* **ResultScene**: (구현 예정) 게임 종료 시 점수를 표시하고 다시 시작 기능을 제공.

### MainScene GameObjects
| Class | 구성 및 동작 정보 | 상호작용 정보 | 핵심 코드 책임 설명 |
| :--- | :--- | :--- | :--- |
| **MJPhoto** | Sprite를 확장한 물리 객체. 중력 가속도 적용. | 터치 위치에서 생성, 바닥/객체와 충돌. | `update()`에서 중력 연산 및 위치 갱신 담당. |
| **World** | Generic Layer 시스템. 4개 레이어 관리. | 객체의 추가/삭제 및 렌더링 순서 제어. | 레이어별로 객체를 분류하여 효율적인 루프 및 드로우 담당. |
| **MainScene** | 게임 씬 자체. 터치 이벤트 리스너 포함. | 화면 터치 시 MJPhoto 객체를 Current 레이어에 생성. | 터치 좌표를 가상 좌표로 변환하여 객체 생성 위치 결정. |

## 7. UX 진행 방법
* **시각적 요소**: `GameMetrics` 가상 좌표계를 사용하여 모든 해상도에서 글자와 이미지가 뚜렷하게 보이도록 설계했습니다.
* **영상 녹화**: 2차 발표 영상(약 1분 45초)에서는 자막과 음성을 활용하여 구현된 핵심 기능을 명확히 설명할 예정입니다.

## 8. 어려웠던 부분 및 건의사항
* **어려웠던 부분**: Gradle 9.0 버전 업그레이드 이후 Android Gradle Plugin과의 호환성 문제로 인해 빌드가 되지 않았던 현상이 있었습니다. AGP 버전을 8.3.0으로 조정하고 필수 리소스 파일들을 수동 생성하여 해결했습니다.
* **추가 학습 희망**: 객체들이 많이 쌓였을 때 발생하는 성능 저하 방지를 위한 최적화 기법이나, 더 정교한 원형/다각형 충돌 물리 엔진에 대해 다루었으면 좋겠습니다.
